# 🚀 빠른 시작 가이드 - AI 코드 후처리

## 4단계로 완성하는 ML 파이프라인

### 1️⃣ 코드 목적 설명하기
우측 사이드바 **"💡 코드 목적 설명"**에 원하는 기능을 입력하세요.

```
예시:
아이리스 데이터로 꽃 종류를 분류하고,
confusion matrix를 그리고,
모델을 파일로 저장하고 싶습니다.
```

---

### 2️⃣ 노드로 기본 흐름 구성하기
좌측에서 노드를 드래그하여 ML 파이프라인을 만드세요.

**추천 흐름**:
```
Data Loader → Data Split → Scaler → Classifier → Evaluate
```

---

### 3️⃣ AI가 코드 완성하기
1. **"🐍 코드 보기"** 버튼 클릭
2. **"✨ AI로 개선하기"** 탭 클릭
3. AI가 자동으로:
   - ✅ 에러 처리 추가
   - ✅ 로깅 및 모니터링
   - ✅ 시각화 코드 생성
   - ✅ 모델 저장 기능
   - ✅ Cross-validation

---

### 4️⃣ 코드 다운로드
- **📓 Jupyter**: `.ipynb` 파일로 저장
- **📄 Python**: `.py` 스크립트로 저장
- AI 개선 버전은 `_ai_enhanced` 접미사가 붙습니다

---

## 💡 핵심 차이점

| 기능 | 노드 기반 코드 | AI 개선 코드 |
|------|----------------|--------------|
| 에러 처리 | ❌ 없음 | ✅ try-except 추가 |
| 시각화 | ❌ 없음 | ✅ matplotlib 그래프 |
| 모델 저장 | ❌ 없음 | ✅ pickle/joblib |
| 로깅 | ❌ 기본만 | ✅ 상세 로깅 |
| 검증 | ❌ 없음 | ✅ Cross-validation |
| 주석 | ⚠️ 최소 | ✅ 한국어 설명 |

---

## 🎯 언제 사용하나요?

### 노드 기반 코드만으로 충분한 경우
- 빠른 프로토타입
- 학습용 코드
- 간단한 실험

### AI 개선이 필요한 경우
- 프로덕션 배포
- 프레젠테이션용
- 에러 처리가 중요한 경우
- 시각화가 필요한 경우
- 모델을 저장해야 하는 경우

## ⚡ 팁

1. **의도를 구체적으로**: "모델 만들기" ❌ → "RandomForest로 분류하고 시각화" ✅

3. **비교하며 배우기**: 원본과 AI 코드를 탭으로 비교하며 학습
4. **여러 번 시도**: 마음에 들지 않으면 의도를 수정하고 재생성
