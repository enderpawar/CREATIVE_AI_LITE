# 🎯 초보자 친화적 노드 설계 (v4.0)

## 핵심 철학: "복잡함을 숨기고, 흐름을 보여주기"

### ❌ 이전 문제점
```
DataSplit → X_train, X_test, y_train, y_test (4개 출력)
Classifier → X_train 입력, y_train 입력 (2개 입력)
```
- 초보자는 X_train, y_train이 뭔지 모름
- 어떤 소켓을 어디에 연결해야 하는지 헷갈림
- ML 지식이 없으면 사용 불가능

### ✅ 개선된 설계
```
DataSplit → "훈련용", "테스트용" (2개 출력)
Classifier → "훈련용" (1개 입력)
```
- 직관적인 한글 라벨
- 대부분의 노드가 1입력 1출력
- 데이터 흐름만 따라가면 됨

---

## 📊 노드별 입출력 구조

### 1️⃣ **Data Loader** (데이터 로더)
```
입력: 없음
출력: 데이터
```
CSV 파일을 불러옵니다.

---

### 2️⃣ **Data Split** (데이터 분할)
```
입력: 데이터
출력: 훈련용, 테스트용
```
데이터를 훈련용과 테스트용으로 나눕니다.

---

### 3️⃣ **Scaler** (스케일러)
```
입력: 데이터
출력: 데이터
```
숫자 범위를 조정합니다.
- 훈련용에 연결하면 훈련용만 처리
- 분할 전 데이터에 연결하면 전체 처리

---

### 4️⃣ **Feature Selection** (특성 선택)
```
입력: 데이터
출력: 데이터
```
중요한 특성만 골라냅니다.

---

### 5️⃣ **Classifier / Regressor** (분류/회귀 모델)
```
입력: 훈련용
출력: 모델
```
데이터를 학습하여 모델을 만듭니다.

---

### 6️⃣ **Neural Network** (신경망)
```
입력: 훈련용
출력: 모델
```
딥러닝 모델을 학습합니다.

---

### 7️⃣ **Predict** (예측)
```
입력: 모델, 테스트용
출력: 예측결과
```
모델로 새로운 데이터를 예측합니다.

---

### 8️⃣ **Evaluate** (평가)
```
입력: 예측결과, 테스트용
출력: 평가결과
```
예측이 얼마나 정확한지 계산합니다.

---

## 🔄 기본 워크플로우

### 분류 모델 만들기
```
Data Loader → Data Split → Classifier → Predict → Evaluate
                   ↓           ↓          ↓
               훈련용       모델      예측결과
                           
                   ↓
               테스트용 ──────────────────────┘
```

### 전처리 추가하기
```
Data Loader → Data Split → Scaler → Classifier → Predict → Evaluate
```

## 🎓 학습 경로

### Level 1: 기본 파이프라인
1. Data Loader로 CSV 불러오기
2. Data Split으로 나누기
3. Classifier로 학습하기

### Level 2: 전처리 추가
1. Scaler로 숫자 조정하기
2. Feature Selection으로 특성 선택하기

### Level 3: 평가 및 튜닝
1. Predict로 예측하기
2. Evaluate로 성능 확인하기
3. Hyperparameter Tuning으로 최적화하기

## 🔑 설계 원칙

1. **1입력 1출력 우선**
   - 대부분의 노드는 입력 1개, 출력 1개
   - 예외: DataSplit (1→2), Predict/Evaluate (2→1)

2. **한글 라벨 사용**
   - "훈련용", "테스트용", "모델", "예측결과"
   - 전문 용어 최소화

3. **직관적인 연결**
   - 색상으로 구분 (데이터: 파란색, 모델: 초록색)
   - 화살표 방향만 따라가면 됨

4. **실수 방지**
   - 잘못된 연결은 자동으로 차단
   - 명확한 에러 메시지 (한글)
